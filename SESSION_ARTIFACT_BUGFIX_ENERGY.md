## Фаза 1: Мерж фикс-ветки и исправление критических багов

**Контекст:** Предыдущая сессия провела полный аудит системы (bot.py, ai_chat.py, admin.py, ai_utils.py) и нашла критические баги. Эта сессия началась с исправления всех найденных проблем.

### Баги из аудита ai_chat.py + admin.py + ai_utils.py (исправлены)

| # | Файл | Серьёзность | Проблема | Решение |
|---|------|-------------|----------|---------|
| 1 | bot.py:3124 | CRITICAL | `callback.answer(url=...)` в ad click handler — работает только для Telegram Games, не для произвольных URL | Заменено на `callback.answer()` + `bot.send_message(uid, ad["url"])` |
| 2 | admin.py:1210 | CRITICAL | AI miss-you SQL: `DISTINCT ON` + `GROUP BY` + `ORDER BY COUNT(*)` — невалидный PostgreSQL, крашит запрос | Переписано на `ROW_NUMBER() OVER (PARTITION BY h.uid ORDER BY COUNT(*) DESC)` через подзапрос |
| 3 | admin.py:612-643 | HIGH | Когортная аналитика SQL: f-string интерполяция в запросах (потенциальная SQL injection) | Заменено на параметризованные запросы с `timedelta` объектами |
| 4 | ai_chat.py:1864 | HIGH | `InputPaidMediaPhoto` для GIF/видео контента — неправильный тип медиа | Заменено на `InputPaidMediaVideo` для hot_gif |
| 5 | admin.py:1533 | MEDIUM | SQL injection в `char_media_view` — поле из callback data шло напрямую в SQL без валидации | Добавлена проверка `field not in _FIELD_LABELS` |

### Баги из аудита bot.py (исправлены)

| # | Файл | Серьёзность | Проблема | Решение |
|---|------|-------------|----------|---------|
| 6 | bot.py:1988 | HIGH | `int(parts[2])` в gift payment — ValueError при некорректном payload крашит обработчик | Добавлен `try/except ValueError` с логированием |
| 7 | bot.py:2052 | MEDIUM | Невалидный `plan_key` в successful_payment молча дефолтился на 1-месячный премиум | Добавлена валидация `plan_key not in PREMIUM_PLANS` с ранним return |
| 8 | bot.py:3082-3092 | HIGH | TOCTOU race condition в рейтинге — два параллельных запроса могли оба пройти SELECT и вставить дубль | Обёрнуто в `async with conn.transaction()` + `FOR UPDATE` lock |

### Что НЕ потребовало исправления (проверено и подтверждено как корректное)
- Voice transcription (`image_url` для audio) — корректный формат для Gemini через OpenRouter
- JPEG MIME type в vision — Telegram всегда конвертирует фото в JPEG
- Admin callback.answer() — все обработчики уже имели вызовы
- Race condition в `relay()` — на практике aiogram обрабатывает апдейты одного юзера последовательно
- `set.discard()` cleanup outside lock — атомарен в Python

### 3.1 Убрана кнопка из главного меню

**Файл:** `keyboards.py` — `kb_main()`

**Было:** 4 ряда кнопок, последний — "Магазин энергии"
**Стало:** 3 ряда. Энергию покупают в контексте (когда кончилась в AI-чате), а не заранее.

### 3.2 Новое ценообразование

**Файл:** `constants.py` — `ENERGY_PACKS`

**Принцип:** Паки — импульсная покупка. Должны быть дороже Premium в пересчёте на единицу, чтобы подталкивать к подписке.

| Пак | Было | Стало | ★/⚡ | Логика |
|-----|------|-------|------|--------|
| Микро | 30⚡ = 49★ | **10⚡ = 29★** | 2.9 | Низкий порог, "ещё чуть-чуть" |
| Средний | 100⚡ = 129★ | **50⚡ = 99★** | 1.98 | Основной пак, целый разговор |
| Большой | 300⚡ = 299★ | **150⚡ = 249★** | 1.66 | Запас, но всё равно дороже Premium |

**Сравнение с Premium:**
- Premium 7d = 129★ → 200⚡/день × 7 = 1400⚡ → **0.09★/⚡**
- Самый дешёвый пак = 150⚡ за 249★ → **1.66★/⚡**
- Разница в 18x мотивирует конверсию в подписку

**Мультипликаторы регионов** (не менялись): RU ×1.0, ES ×1.3, EN ×2.0

## Архитектура энергии (итоговая)

```
Юзер пишет AI-персонажу
    ↓
ai_chat.py проверяет энергию:
  - cost = ENERGY_COST[char_tier] (1/2/3)
  - max = DAILY_ENERGY[user_tier] + ai_bonus
  - used = user.ai_energy_used
  - Если used + cost > max → показать "энергия кончилась" + магазин
    ↓
Покупка энергии:
  - Кнопка "Пополнить энергию" → callback "energy_shop"
  - energy_shop.py показывает магазин с баром и таймером
  - Юзер выбирает пак → Telegram Stars invoice
  - После оплаты: ai_energy_used -= pack.amount (в bot.py successful_payment)
    ↓
Или Premium:
  - CTA "Premium — 200⚡/день" → callback "premium_show"
  - Premium даёт 200⚡/день вместо 30⚡
```

### Файлы системы энергии:
- `constants.py` — ENERGY_PACKS, DAILY_ENERGY ценообразование
- `energy_shop.py` — UI магазина (callback handlers)
- `ai_chat.py` — проверка и расход энергии, `get_energy_info()`, `_energy_bar()`
- `bot.py` — обработка оплаты (`successful_payment`), текстовый хендлер магазина
- `keyboards.py` — `kb_energy_shop()` с кнопками паков + Premium CTA
- `locales.py` — все тексты энергии на RU/EN/ES

## Ключевые решения и обоснования

1. **Почему убрали кнопку энергии из меню** — энергия нужна только в контексте AI-чата. Кнопка в главном меню увеличивает когнитивную нагрузку и не приводит к конверсии (юзер не покупает энергию заранее).

2. **Почему паки стали меньше и дороже за единицу** — паки должны быть импульсной покупкой "мне нужно прямо сейчас", а Premium — рациональным выбором "это выгоднее". Если паки дешевле Premium на единицу, нет мотивации подписываться.

3. **Почему CTA Premium в магазине энергии** — юзер пришёл покупать энергию = уже готов платить. Показать ему что подписка в 18x выгоднее = высокая конверсия.

4. **Почему визуальный бар и таймер** — бар создаёт ощущение ресурса (loss aversion), таймер — urgency. Вместе работают на конверсию.

5. **Почему `callback.answer(url=...)` заменили на сообщение** — Telegram Bot API `answerCallbackQuery` с параметром `url` работает только для Game URLs (документация Telegram). Для обычных ссылок нужно отправлять сообщением.

6. **Почему транзакция в рейтинге** — classic TOCTOU: два параллельных запроса проходят SELECT, оба не находят существующий рейтинг, оба делают INSERT. Транзакция с FOR UPDATE блокирует второй запрос до завершения первого.
