## TL;DR

Разработан с нуля анонимный Telegram-дейтинг бот с 13 AI-персонажами, монетизацией через Telegram Stars и поддержкой трёх языков. Готовая к продакшну система на 100K+ MAU с двухфазным алгоритмом матчинга, долгосрочной AI-памятью и отдельным ботом для администрирования.

## О проекте

**MatchMeBot** — анонимный мессенджер-дейтинг внутри Telegram. Пользователи общаются в трёх режимах: простое общение, флирт, кинк. Если партнёр не найден — бот предлагает чат с одним из 13 AI-персонажей. Монетизация через Premium-подписки, пачки энергии и подарки — всё через Telegram Stars.

| Режимы общения | Simple · Flirt · Kink |
| AI-персонажей | 13 (3 тира) |
| SKU (продукты) | 9 (3 подписки + 3 энергии + 3 подарка) |
| Строк кода | ~19 000 (main + admin) |

| FSM-состояний | 30+ |

## Архитектура

### Модульная структура (6 роутеров)

Монолит в 3 400 строк был разбит на 6 независимых модулей с Dependency Injection через `bot.py`. Роутеры не импортируют друг друга — единственная связь идёт через callback-функции, зарегистрированные при старте.

```
bot.py              → точка входа, DI-регистратор, мониторинг
registration.py     → онбординг FSM: name → age → gender → mode → interests
matching.py         → двухфазный поиск партнёра
chat.py             → relay-чат, лайки, рейтинг, подарки, жалобы
profile.py          → профиль, уровни, стрики, квесты, достижения
payments.py         → Premium, Telegram Stars инвойсы
ai_chat.py          → AI-чат, энергия, долгосрочная память
```

### Двухфазный матчинг

Ключевая оптимизация для масштаба — вместо O(N) запросов к БД:

1. **Redis фаза:** `SRANDMEMBER` → до 50 случайных кандидатов из очереди
2. **DB фаза:** один `SELECT WHERE uid = ANY($1)` для всех кандидатов сразу
3. Атомарное спаривание через Lua-скрипт (защита от race condition)

**Результат:** 1 000 человек в очереди → время матча 20 мс (против 2 000 мс при наивном подходе).

### Redis-состояние

```
mm:chat:active:{uid}        → partner_uid
mm:queue:{mode}:{premium}   → UIDs в поиске (7 очередей)
mm:ai:session:{uid}         → хэш сессии AI (TTL 30 мин)
mm:ai:memory:{uid}:{char}   → долгосрочная память (TTL 30 дней)
mm:ai:facts:{uid}:{char}    → до 10 фактов о пользователе
mm:ratelimit:{uid}:{action} → sliding window rate limit
```

При недоступном Redis — автоматический fallback на Python dicts.

## AI-система

### 13 персонажей в 3 тирах

| Тир | Модель | Персонажи | Стоимость |
|-----|--------|----------|----------|
| **Basic** | GPT-4o-mini | Luna, Max, Mia, Kai | 1 ⚡/сообщение |
| **VIP** | Gemini Flash / Hermes-70B | Aurora, Alex, Diana, Leon | 2 ⚡/сообщение |
| **VIP+** | Hermes-405B | Lilit, Eva, Damir, Ars, Master | 3 ⚡/сообщение |

Каждый персонаж: детальный system prompt, уникальный голос, медиа-контент (фото/GIF, включая blurred-версии), ролевые правила.

### Долгосрочная память

- По завершении сессии (≥4 сообщений) — Gemini Flash суммаризирует диалог
- Хранятся последние 2 резюме + до 10 фактов о пользователе (TTL 30 дней)
- Загружаются при старте новой сессии → персонаж «помнит» прошлые разговоры
- Кэшируются в session hash — экономит 2 Redis-вызова на каждое сообщение

### Budget control

- Часовой лимит AI-запросов: 500 (настраивается через `AI_HOURLY_LIMIT`)
- При превышении — fallback на Gemini Flash
- Логирование token usage из ответов OpenRouter

### Unit Economics

- **Маржа Premium 1м (kink, максимальный AI-расход):** 87.7%
- **Break-even:** ~100 пользователей при конверсии 3%
- **ARPU (RU, 7% платящих):** $0.44/мес (с учётом energy + gifts)
- **Маржа при 10K юзерах / 7% конверсии:** ~90%

### Механики удержания и монетизации

- **A/B тестирование:** группа B получает скидку 15% на Premium
- **Trial Premium:** одноразовое предложение после 5-го чата
- **Система подарков:** роза/бриллиант/корона → дарит получателю Premium на 1/3/7 дней
- **Квесты:** 3 случайных из 5 типов ежедневно → +5 бонусной энергии за выполнение всех
- **Уровни:** 6 уровней по `total_chats`, разблокируют режимы
- **Return gifts:** кампания возврата неактивных пользователей с подарком-стимулом
- **Streak:** ежедневный бонус за непрерывное использование

## Admin Bot (отдельный процесс)

4 555 строк. Работает независимо от основного бота, общается через таблицу `bot_stats` в PostgreSQL.

- **Канал:** автогенерация постов (истории, шутки, опросы) через AI
- **Модерация постов:** рабочий процесс approve / reject / regen с FLUX.2 Pro для изображений
- **Управление пользователями:** бан, shadow ban, статистика, когортный анализ
- **Маркетинг:** A/B группы, кампании возврата, напоминания о стриках
- **Поддержка:** обработка жалоб, эскалация проблем

## Ключевые технические решения

### Проблема → Решение

| Проблема | Решение |
|---------|---------|
| Циклический импорт между роутерами | Callback-регистрация через `bot.py` при инициализации |
| Race condition при матчинге | Атомарный Lua-скрипт `PAIR_SCRIPT` |
| 2 лишних Redis-вызова на каждое AI-сообщение | Кэш в session hash (`loaded_memory`) |
| Пропуск переводов при добавлении ключей | Единый `locales.py` + `_all(key)` для FSM-фильтров |
| Рост AI-затрат при пиках трафика | Часовой лимит + fallback на дешёвую модель |
