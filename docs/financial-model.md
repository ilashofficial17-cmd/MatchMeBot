## 3. Unit Economics

### 3.1. CAC по каналам

| Канал | CPA диапазон | CPA среднее | Валюта |
|-------|-------------|-------------|--------|
| РУ посевы (Telegram) | 30-150₽ | 100₽ ($1.11) | ₽ |
| EN (Meta/Google) | $2-8 | $5.00 | $ |
| ES (Meta/TikTok) | $0.50-2.50 | $1.50 | $ |

### 3.2. ARPU (Average Revenue Per User)

При 7% платящих (бенчмарк dating):

| Регион | Premium 1м Stars | Чистый доход | ARPU/мес (× 7%) |
|--------|-----------------|-------------|-----------------|
| RU | 349 | $4.89 | **$0.34** |
| ES | 454 | $6.36 | **$0.45** |
| EN | 698 | $9.77 | **$0.68** |

С учётом Energy + Gifts (оценка +30% к Premium revenue):

| Регион | ARPU/мес (total) |
|--------|-----------------|
| RU | **$0.44** |
| ES | **$0.58** |
| EN | **$0.89** |

### 3.3. LTV при разных retention rates

Формула: `LTV = ARPU × (1 / (1 - retention_rate))`

Simplified: `LTV = ARPU × avg_months_active`

| Регион | Retention 1м (отток 80%) | Retention 3м (отток 50%) | Retention 6м (отток 30%) |
|--------|-------------------------|-------------------------|-------------------------|
| RU | $0.44 × 1.25 = **$0.55** | $0.44 × 2.0 = **$0.88** | $0.44 × 3.3 = **$1.45** |
| ES | $0.58 × 1.25 = **$0.73** | $0.58 × 2.0 = **$1.16** | $0.58 × 3.3 = **$1.91** |
| EN | $0.89 × 1.25 = **$1.11** | $0.89 × 2.0 = **$1.78** | $0.89 × 3.3 = **$2.94** |

### 3.4. Break-even: LTV > CAC

| Регион | CAC | LTV (3м retention) | LTV/CAC | Окупается? |
|--------|-----|--------------------|---------|-----------:|
| RU | $1.11 | $0.88 | 0.79 | **Нет** (при 7%) |
| ES | $1.50 | $1.16 | 0.77 | **Нет** (при 7%) |
| EN | $5.00 | $1.78 | 0.36 | **Нет** (при 7%) |

**При каком % платящих окупается?**

Формула: `min_paying% = CAC / (revenue_per_payer × avg_months)`

| Регион | CAC | Rev/payer/мес | Нужно % (3м ret) | Нужно % (6м ret) |
|--------|-----|--------------|-------------------|-------------------|
| RU | $1.11 | $6.36 | **8.7%** | **5.3%** |
| ES | $1.50 | $8.27 | **9.1%** | **5.5%** |
| EN | $5.00 | $12.70 | **19.7%** | **11.9%** |

**Вывод:**
- **RU и ES окупаются при 6-месячном retention и 5-6% платящих** — реалистично
- **EN не окупается** при платном привлечении — нужен органический рост или снижение CPA
- При CPA 30₽ (нижняя граница РУ): окупается при **2.6% платящих** — легко

## 6. Функция get_price()

### Текущая реализация (фактический код)

```python
# constants.py (уже есть):
PRICE_MULTIPLIERS = {"ru": 1.0, "es": 1.3, "en": 2.0}

# energy_shop.py:92 (уже есть):
mult = PRICE_MULTIPLIERS.get(lang, 2.0)
price = int(pack["stars"] * mult)
```

### Рекомендуемая единая функция

```python
# constants.py — добавить:
def get_price(product_key: str, lang: str, ab_group: str = "A") -> int:
    """Единая точка расчёта цены в Stars."""
    # Определяем базовую цену
    if product_key in PREMIUM_PLANS:
        base = PREMIUM_PLANS[product_key]["stars"]
    elif product_key in ENERGY_PACKS:
        base = ENERGY_PACKS[product_key]["stars"]
    elif product_key in GIFTS:
        base = GIFTS[product_key]["stars"]
    else:
        raise ValueError(f"Unknown product: {product_key}")

    # Региональный множитель
    mult = PRICE_MULTIPLIERS.get(lang, 2.0)

    # A/B тест скидка
    if ab_group == "B":
        mult *= AB_PRICE_DISCOUNT_B  # 0.85

    return max(1, int(base * mult))
```

**Файлы для рефакторинга:**
- `bot.py` — заменить ручные расчёты Stars на `get_price()`
- `energy_shop.py:92` — заменить на `get_price(pack_key, lang)`

## 7. Рекомендации

### 7.1. Ценовая стратегия для старта

**Не менять текущие цены.** Они адекватны:
- RU: конкурентоспособны (дешевле Tinder)
- EN: есть запас для роста
- ES: середина рынка

### 7.2. Когда повышать цены

| Триггер | Действие |
|---------|----------|
| Конверсия > 10% | Поднять RU на 10% (тест через A/B) |
| EN юзеров > 2000 | Поднять EN множитель до 2.5× |
| Retention 6м > 40% | Добавить годовой план (выгоднее для юзера, LTV растёт) |

### 7.3. Free tier AI — оставить или убрать?

**Оставить Free tier (30 энергии/день).** Обоснование:

| Без Free AI | С Free AI (текущее) |
|------------|---------------------|
| Конверсия в платящих выше (+2-3%) | Конверсия ниже |
| Retention значительно ниже (-40%) | Retention выше (юзер вовлечён) |
| AI cost = $0 для free | AI cost = $0.05/юзер/мес |
| Отток юзеров на конкурентов | Юзер привыкает → покупает |

**При 1000 юзерах:** Free AI стоит ~$35/мес. Это маркетинговый расход — дешевле любого CPA канала.

### 7.4. Ограничение бесплатных чатов

**Не ограничивать анонимные чаты.** Это core-функция, без неё бот мёртв.

**Что ограничивать для монетизации:**
- VIP+ персонажи (уже закрыты для Free)
- AI сообщений в день (уже лимит 30 энергии)
- Продвинутые фильтры поиска (возможная будущая фича)

### 7.5. Годовой план

Добавить при достижении 5K юзеров:
```python
"1y": {"stars": 2499, "days": 365, "label": "plan_label_1y", "desc": "plan_desc_best", "tier": "premium"}
```
RU: 2499 Stars = $49.98/год = $4.17/мес (40% скидка от месячного).
LTV с годовым планом: $35/год чистых — покрывает CAC мгновенно.

## 8. Итоговая формула рентабельности

```
Monthly Profit = (Users × PayingRate × AvgRevenuePerPayer × 0.70)
               - (DAU × AvgAICostPerUser)
               - FixedCosts

Где:
  AvgRevenuePerPayer = get_price("1m", weighted_lang) × 1.3 (energy+gifts)
  0.70 = (1 - 30% Telegram commission)
  DAU = Users × 0.30
  AvgAICostPerUser = $0.133/мес
  FixedCosts = $20/мес
```

**Break-even (минимум юзеров для прибыли):**
```
При 3% конверсии, RU:
  Revenue/user/мес = 0.03 × $6.36 × 1.3 = $0.248
  Cost/user/мес = 0.30 × $0.133 = $0.040
  Net/user/мес = $0.208
  Break-even = $20 / $0.208 = 97 юзеров

→ Проект выходит в плюс с ~100 юзеров при 3% конверсии.
```
