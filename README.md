# docs/session-bible — База знаний и артефакты сессий

> **Эта ветка не содержит кода.** Только документация: артефакты сессий с Claude и проектная библия.
> Код проекта — в ветке `main`.

---

## Зачем эта ветка

Каждая сессия с Claude Code — отдельный контекст. Когда сессия заканчивается, Claude забывает всё.
Эта ветка решает проблему: в конце каждой сессии сюда загружается артефакт — полный конспект того, что обсуждалось и делалось. Следующая сессия читает артефакты и сразу входит в контекст.

## Как пользоваться

### Начало новой сессии
Скажи Claude в начале сессии:

```
Прочитай артефакты из ветки docs/session-bible — там контекст проекта.
Начни с PROJECT_BIBLE.md, потом самый свежий SESSION_ARTIFACT_*.md
```

### В конце сессии (или по ходу)
Скажи Claude:

```
Создай артефакт этой сессии и загрузи его в ветку docs/session-bible
```

Claude сам:
1. Создаст файл `SESSION_ARTIFACT_<тема>.md`
2. Запишет что обсуждалось, что изменено, все коммиты
3. Запушит в эту ветку

## Формат артефакта сессии

Каждый артефакт должен содержать:

```markdown
# Артефакт сессии — <дата> (сессия N)
> Краткое описание темы

## Исходный контекст
Что было на входе, какие проблемы решались

## Фаза 1: <тема>
### Изменения по файлам
Таблица: что | было | стало

## Фаза N: ...

## Итоговая схема / архитектура (если применимо)
Таблицы, схемы, решения

## Все коммиты сессии
| Хэш | Описание |

## Важные архитектурные решения
Список решений которые важно помнить в будущих сессиях

## Правила для следующих сессий (опционально)
Поведенческие паттерны которые нужно соблюдать
```

---

## Правила ветки

- **Никакого кода** — только `.md` файлы
- **Не мержить в main** — ветка изолирована
- **Не удалять старые артефакты** — это история проекта
- **Называть файлы по теме**, не по дате: `SESSION_ARTIFACT_<ТЕМА>.md`
- `PROJECT_BIBLE.md` — единственный файл без префикса `SESSION_ARTIFACT_`
